# VeloIreland - Cycling Route Guide

An Angular + PrimeNG app for showcasing cycling routes, deployable on GitHub Pages.

> This file is generated by `python update_readme.py`. Run `npm run readme:update` after changing route data, or use the npm scripts below that do it automatically.

## Features

- Dynamic JSON content loaded from `public/routes.json`
- Route data generated from GPX files in `src/assets/gpx_files`
- Long Routes / Short Routes tabs
- Route cards with map image, elevation profile, distance, ascent, max altitude, description, and expanded details
- Responsive layout using Angular and PrimeNG

## Updating Routes And README

Add or edit GPX files in:

- `src/assets/gpx_files/longRoutes`
- `src/assets/gpx_files/shortRoutes`

Then run:

```bash
npm run routes:update
```

That command refreshes `public/routes.json` and regenerates this README. To preview route changes without writing JSON:

```bash
npm run routes:preview
```

To only refresh the README:

```bash
npm run readme:update
```

To generate missing route descriptions with OpenAI, create a local `.env` file in the project root:

```bash
OPENAI_API_KEY=sk-your-key-here
```

Then run:

```bash
npm run routes:update:copy
```

## Commands

| Command | Purpose |
| --- | --- |
| `npm install` | Install project dependencies. |
| `npm start` | Start the local Angular dev server. |
| `npm run routes:update` | Refresh route data from GPX files. |
| `npm run routes:update:copy` | Refresh routes and fill missing descriptions with OpenAI. |
| `npm run routes:preview` | Preview GPX route changes without writing routes.json. |
| `npm run readme:update` | Regenerate this README from project metadata. |
| `npm run build` | Build the Angular app. |
| `npm run build:gh-pages` | Build the GitHub Pages version. |
| `npm run deploy` | Build and publish the site to the gh-pages branch. |

## Local Development

```bash
npm install
npm start
```

Open `http://localhost:4200`.

## Deploy To GitHub Pages

```bash
npm run deploy
```

The deploy workflow refreshes this README, builds the GitHub Pages version, and publishes `dist/cycling-routes/browser` to the `gh-pages` branch.

After the first deploy, go to **Settings -> Pages -> Branch: gh-pages -> Save**.

If GitHub Pages keeps serving an older version after a successful deploy, force a fresh `gh-pages` commit:

```bash
npx gh-pages -d dist/cycling-routes/browser --message "Force GitHub Pages redeploy"
```

Then wait a minute and check `https://<username>.github.io/<repo-name>/routes.json`.

Your site URL will be:

```text
https://<username>.github.io/<repo-name>/
```
